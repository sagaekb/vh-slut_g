Examensarbete Versionshantering/h1> terminologi

Versionshantering är ett sätt att hålla reda på och hantera ett projekts olika versioner. Med detta menas det att genom ett verktyg, som exempelvis GitHub, kan en eller flera personer som arbetar med projektet se alla ändringar och på ett enkelt sätt navigera sig mellan dessa olika ändringar - alltså versioner. En klar fördel med detta är att det är enklare att hålla reda på exakt vad som har skett i ett projekt och dess versioner samt att flera användare kan hantera ett projekt samtidigt. Det kan göras genom att man använder sig av Git repository. Repository innehåller ett projekts alla filer och versioner så att användarna som ovan nämnt kan hålla reda på allt i projektet. En nackdel som kan knytas in i detta är att det kan bli rörigt med de olika versionerna och att hålla koll på dem. I samband med detta kan konflikter uppstå. En konflikt kan exempelvis vara att flera ändringar av samma del av ett projekt kan ändras samtidigt vilket då kan bli problematiskt. Detta kallas även för merge conflict. Merge är precis som det betyder på engelska, att man slår ihop, i detta sammanhang innebär det att man slår ihop delar av arbetet så att det blir en och samma del.
